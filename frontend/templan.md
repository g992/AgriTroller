## План конструктора устройств (MVP)

- **Типы устройств**: справочник (`device_types`) с полями sensors, actuators, error_fields, settings_schema, mapping_defaults; встроенные generic/htl + CRUD для кастомных.
- **Настройка устройства**: для каждого устройства выбираем тип, задаём порт/baudrate, сохраняем mapping (pipeline операций: hex_to_int, scale, offset, mask, round, to_int, to_float) и значения settings_schema.
- **Modbus-сканнер**: выбор RS485 адаптера и диапазона адресов + регистр/функция, отображение прогресса и результатов; найденные устройства можно быстро привязать к типу (создать новое устройство или обновить существующее).
- **UI конструктора**:
  1. Панель “Типы устройств”: список, создание кастомного типа (имя/slug, поля сенсоров/актуаторов, ошибки, схема настроек, дефолтный mapping).
  2. Панель “Сканнер Modbus”: форма запуска, прогресс, результаты, выбор типа для найденных адресов, создание устройства.
  3. Панель “Устройства”: список устройств с типом и mapping; форма редактирования (порт, скорость, тип, mapping, настройки устройства).
- **Ограничения сканера**: диапазон адресов 1-247, функции чтения (0x03/0x04), регистр 0-65535, шаг по одному регистру, таймаут на запрос; прогресс в процентах, сбор found[] (адрес, регистр, значение, raw hex).
- **Диагностика/ошибки**: error_fields типов для хранения кодов/флагов датчика, показ отдельной секцией.
